# 日报组装 Prompt

你正在将各板块的摘要组装成一份完整的 AI 日报。

## 核心理念

日报以**事件**为核心，不以信息源为核心。读者关心的是"今天发生了什么"，而不是"每个源说了什么"。同一事件被多个渠道报道时，聚合为一条，展示官方/媒体/社区的不同视角。

## 格式

以此标题开头（替换日期）：

```
# AI 日报 · YYYY-MM-DD

> 采集时间 HH:MM | 信息源: X | 条目: X
```

## 板块顺序

1. **今日观察** — 从 3-5 位不同领域专家的视角总结今日要点（见下方格式）
2. **重大新闻** — 当天最重要的 3-5 个事件，跨所有来源选出，使用多源聚合格式
3. **公司动态** — AI 公司的产品发布、战略调整、人事变动等
4. **Twitter/X 精选** — 来自监控清单的重要推文（不与重大新闻重复）
5. **论文与开源** — arXiv 论文、GitHub Trending、HuggingFace
6. **社区热点** — HackerNews、Reddit、V2EX 等（不与重大新闻重复）

每个板块使用 `## 板块名` 作为标题。

## 今日观察格式

从不同领域的知名人物视角出发，用他们的口吻和关注点总结今日要点。每位专家写一段 2-3 句话的点评。

选择专家时优先从以下人物中挑选与当天热点最相关的 3-5 位：
- 马斯克（Elon Musk）— 关注 AI 安全、开源、商业竞争
- 贝佐斯（Jeff Bezos）— 关注基础设施、云计算、商业模式
- 黄仁勋（Jensen Huang）— 关注算力、芯片、GPU
- 奥特曼（Sam Altman）— 关注 AGI 进展、OpenAI 动态
- Dario Amodei — 关注 AI 安全、对齐、Anthropic
- 扎克伯格（Mark Zuckerberg）— 关注开源模型、Meta AI
- 纳德拉（Satya Nadella）— 关注企业 AI、微软生态
- 李飞飞（Fei-Fei Li）— 关注学术研究、AI 伦理
- 李彦宏 — 关注中国 AI 产业、百度

格式：

```
## 今日观察

### 专家名（英文名）

用该专家的视角和关注点，对今天最值得关注的事件进行 2-3 句话的点评。语气要符合该人物的一贯风格。

### 另一位专家

...
```

注意：不要编造专家未说过的话，这是**模拟视角**，不是引用。内容基于当天采集到的真实事件。

## 多源聚合格式（重大新闻必须使用）

当同一事件有多个信息源时：

```
### 事件标题

**来源**:
- 官方: [来源名](url)
- 媒体: [来源名](url) · [来源名](url)
- 社区: [来源名](url)

**官方公告**: ...
**媒体补充**: ...
**社区反应**（如有）: ...

> 影响评估: 一句话
```

详细规则见 `prompts/summarize-news.md`。

## 单源格式（其他板块可使用）

```
### 标题

**来源**: [来源名](url)

事件描述，100-200字。
```

## 去重和归属规则

- **重大新闻中已出现的事件**，不要在其他板块重复。例如 OpenAI 融资如果放在"重大新闻"，就不要在"公司动态"再写一遍
- **Twitter 中提到的事件如果已在重大新闻中聚合**，Twitter 板块跳过该事件，只保留独立观点/未被聚合的推文
- **HN/Reddit 讨论如果已在重大新闻中作为"社区反应"出现**，社区板块跳过

## 强制规则

- **没有链接的内容不收录**
- **不要编造** — 所有信息必须来自采集数据
- **不要模糊概括** — 每条摘要要有具体的数字、人名、产品名
- **多源事件的所有链接都列出** — 不要只保留一个源
- 如果某个板块没有内容，**跳过该板块**
- Twitter 板块如果采集失败，注明"Twitter 数据未采集（opencli 未连接）"

## 结尾

在最后添加采集统计表格：

```
---

## 采集统计

| 指标 | 数值 |
|------|------|
| 信息源总数 | X |
| 成功采集 | X |
| 条目总数 | X |
| 去重移除 | X |
```
