# 日报组装 Prompt

你正在将各板块的摘要组装成一份完整的 AI 日报。

## 核心理念

日报以**事件**为核心，不以信息源为核心。读者关心的是"今天发生了什么"，而不是"每个源说了什么"。同一事件被多个渠道报道时，聚合为一条，展示官方/媒体/社区的不同视角。

## 格式

以此标题开头（替换日期）：

```
# AI 日报 · YYYY-MM-DD

> 采集时间 HH:MM | 信息源: X | 条目: X
```

## 板块顺序

1. **今日观察** — 一段连贯的综合解读，帮助读者快速理解今天 AI 领域发生了什么（见下方格式）
2. **重大新闻** — 当天最重要的 3-5 个事件，跨所有来源选出，使用多源聚合格式
3. **公司动态** — AI 公司的产品发布、战略调整、人事变动等
4. **Twitter/X 精选** — 来自监控清单的重要推文（不与重大新闻重复）
5. **论文与开源** — arXiv 论文、GitHub Trending、HuggingFace
6. **社区热点** — HackerNews、Reddit、V2EX 等（不与重大新闻重复）

每个板块使用 `## 板块名` 作为标题。

## 今日观察格式

写一段 150-300 字的**连贯文本**，综合解读今天 AI 领域最值得关注的趋势和事件。

要求：
- **一整段文字**，不分人物、不分小节、不用 H3 标题
- 融合多个视角（安全、商业、技术、产业、开源等），但**不要提及任何具体人物名字**
- 不要出现"从 XX 的角度来看""以 XX 的视角"这种措辞
- 帮助读者理解今天这些新闻背后的脉络和趋势，而不只是罗列事件
- 语气专业但易读，像一位资深科技记者的综述
- 内容基于当天采集到的真实事件，不要编造

格式：

```
## 今日观察

一整段连贯的综合解读文本，150-300 字。涵盖今天最重要的趋势线索和事件之间的关联。
```

## 多源聚合格式（重大新闻必须使用）

当同一事件有多个信息源时：

```
### 事件标题

**来源**:
- 官方: [来源名](url)
- 媒体: [来源名](url) · [来源名](url)
- 社区: [来源名](url)

**官方公告**: ...
**媒体补充**: ...
**社区反应**（如有）: ...

> 影响评估: 一句话
```

详细规则见 `prompts/summarize-news.md`。

## 单源格式（其他板块可使用）

```
### 标题

**来源**: [来源名](url)

事件描述，100-200字。
```

## 去重和归属规则

- **重大新闻中已出现的事件**，不要在其他板块重复。例如 OpenAI 融资如果放在"重大新闻"，就不要在"公司动态"再写一遍
- **Twitter 中提到的事件如果已在重大新闻中聚合**，Twitter 板块跳过该事件，只保留独立观点/未被聚合的推文
- **HN/Reddit 讨论如果已在重大新闻中作为"社区反应"出现**，社区板块跳过

## 强制规则

- **没有链接的内容不收录**
- **不要编造** — 所有信息必须来自采集数据
- **不要模糊概括** — 每条摘要要有具体的数字、人名、产品名
- **多源事件的所有链接都列出** — 不要只保留一个源
- 如果某个板块没有内容，**跳过该板块**
- Twitter 板块如果采集失败，注明"Twitter 数据未采集（opencli 未连接）"

## 结尾

在最后添加采集统计表格：

```
---

## 采集统计

| 指标 | 数值 |
|------|------|
| 信息源总数 | X |
| 成功采集 | X |
| 条目总数 | X |
| 去重移除 | X |
```
